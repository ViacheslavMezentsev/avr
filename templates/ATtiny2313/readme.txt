                     И Н С Т Р У М Е Н Т Ы

1. Enterprise Architect - средство визуального моделирования на основе
        языка UML 2. Предназначено для автоматизированного формирования
        структуры классов программы.

2. Microsoft Visual Studio 2008/2010 - средство разработки программного кода
        на языке C++. Visual Studio 2008 связана с EA специальным дополнением.

3. WinAVR - набор средств для компиляции исходного кода при помощи
        свободного компилятора AVR GCC.

4. MHV AVR Tools - набор средств для компиляции исходного кода при помощи
        свободного компилятора AVR GCC (замена для WinAVR).
        
5. IAR - средство разработки программного кода на языке C++. В IAR также
        удобно отлаживать проект как в симуляторе, так и внутрисхемно.

6. AVR Studio - средство разработки программного кода на ассемблере и
        GNU C++. При помощи IAR можно создать проект для AVR Studio, в
        котором также можно отлаживать программный код.

7. Proteus - средство визуального моделирования макета устройства.

8. HappyJTAG2 2.45 - простой отладчик для AVR контроллеров, эмулирующий
   работу JTAG ICE MkII, умеет прошивать и по SPI.

9. Bascom AVR - средство разработки программного кода на бейсике.

10. PrimalScript - IDE для разработки скриптов Windows Script Host и
        многого другого.

11. Subversion - VCS (Version Control System) -  свободная централизованная
        система управления версиями.

12. TortoiseSVN - бесплатный клиент для системы контроля версий Subversion,
        выполненный как расширение оболочки Windows и распространяемый
        под лицензией GPL.

13. VisualSVN - средство для удобного управления хранилищем, дополнение
    к VS2008 (по желанию).

14. Visual Basic 6 - простая среда разработки для Windows.

15. Virtual Serial Ports Driver XP 4.0 - утилита позволяет эмулировать
    последовательный порт RS232, соединенный виртуальным нуль-модемным
    кабелем.
    
16. ZOC/Pro 5.07 - эмулятор терминала и telnet/ssh клиент.

Команды для сборки проекта (для VS2008):

Для MHV AVR Tools 20131017 (binutils скопировать из WinAVR):

build: C:\MHVAVRTools\utils\bin\make.exe build
clean: C:\MHVAVRTools\utils\bin\make.exe clean
build all: C:\MHVAVRTools\utils\bin\make.exe build all

Для того, чтобы можно было в VS2008 (или аналогичных) автоматически прыгать на
строку с ошибкой, нужно пропустить вывод через потоковый фильтр (sed). Эта операция
корректирует вывод gcc компилятора, чтобы VS2008 могла его понять:

build: C:\MHVAVRTools\utils\bin\make.exe build 2>&1 | C:\MHVAVRTools\utils\bin\sed.exe -e 's/\(\w\+\):\([0-9]\+\):/\1(\2):/'
clean: C:\MHVAVRTools\utils\bin\make.exe clean
build all: C:\MHVAVRTools\utils\bin\make.exe build all 2>&1 | C:\MHVAVRTools\utils\bin\sed.exe -e 's/\(\w\+\):\([0-9]\+\):/\1(\2):/'

Примечание:
1. Пути к утилите make нужно указывать явно, т.к. по известным
путям таких утилит может быть несколько для разных систем.
Добавьте пути поиска включаемых файлов в свойствах проекта, если
переход по ним через контекстное меню редактора кода не работает.
Убедитесь, что в значении системной переменной PATH имеется путь до утилит AVR GCC.
Это можно сделать, вызвав команду "set path" в командном интерпретаторе (cmd.exe)

2. Если возникает ошибка: avr-gcc: CreateProcess: No such file or directory
Скорее всего у вас установлена два набора компиляторов: WinAVR и  Atmel  Studio
6. Нужно выбрать какой-то один вариант и для него прописать первые пути доступа
в переменной окружения PATH.

3.  Если возникает ошибка: error PRJ0003 : Error spawning 'cmd.exe'
Нужно проверить наличие пути C:\Windows\system32 в настройках VS:
Tools -> Options -> Projects and Solutions -> VC++ Directories (Executable Files)

.cpp - файлы исходный текстов на языке C++;
.h - заголовочные файлы;
.rc - файл ресурсов (разного формата);
Makefile - файл сценария утилиты сборки проекта make (avrgcc).
